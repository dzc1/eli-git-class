Docstring-to-SDK Documentation Pipeline

Extract docstrings and type annotations from Python/JavaScript code to automatically generate comprehensive SDK reference documentation, organized by module and version.

Skill Name: docstring-extractor Priority: P1 Timeline: Month 2 (Phase 1)

How It Works

  1. Skill scans Python/JavaScript repositories for function/class definitions
  2. Extracts docstrings and type annotations from code
  3. Uses AI to enhance descriptions, add usage examples, and improve clarity
  4. Generates SDK reference documentation organized by module
  5. Publishes versioned documentation (e.g., SDK v2.1.0 docs) to Confluence

## `create_booking()`

Creates a new hotel booking for the specified hotel and dates.

**Parameters:**
- `hotel_id` (string, required): Unique identifier for the hotel
- `check_in` (date, required): Check-in date in YYYY-MM-DD format
- `check_out` (date, required): Check-out date in YYYY-MM-DD format

**Returns:** Booking object containing confirmation number, status, and pricing

**Example Usage:**
```python
booking = create_booking(
    hotel_id="HTL12345",
    check_in=date(2026, 3, 15),
    check_out=date(2026, 3, 20)
)
print(booking.confirmation_number)
```

Benefits

Key Outcomes

SDK documentation generated from source code · Developers write docstrings once, documentation auto-creates · Reduces manual SDK documentation by 85% · Version-aware (SDK v1 docs vs v2 docs based on Git tags) · Eliminates disconnect between code and documentation

  • Single Source of Truth: SDK docstrings drive documentation; changes are reflected on next generation
  • Developer Productivity: Engineers write docstrings during development; SDK docs are free artifact
  • Version Management: Automatically generates separate docs for each SDK version (v1, v2, v3)
  • Consistency: All SDK functions documented in standardised format with examples
  • Reduced Maintenance: No manual doc updates required when code changes
  • Improved Onboarding: Clear examples reduce support questions from SDK users

Tooling Integration

The pipeline integrates with industry-standard code analysis tools:

  • Code Parsing: Sphinx (Python), JSDoc (JavaScript), Rustdoc (Rust) for extracting docstrings
  • AST Analysis: Abstract Syntax Tree parsers for understanding code structure and relationships
  • AI Enhancement: Natural language polish and example generation using GPT-4 or Claude
  • Versioning: Git tag detection for version-specific documentation sets
  • Publication: Confluence REST API for automatic publishing to documentation spaces

Example Workflow

Scenario: Publishing SDK v2.1.0 Documentation

When a Git tag v2.1.0 is pushed to the SDK repository:

  1. Skill checks out the repository at that specific tag
  2. Parses all Python/JavaScript files for docstrings and type hints
  3. Extracts method signatures, parameters, return types
  4. Uses AI to generate examples if not provided
  5. Generates module-level documentation pages
  6. Publishes to Confluence under "SDK Documentation v2.1.0" space
  7. Tags all pages with version metadata (version: 2.1.0, ai-generated: true)

Quality Validation

Before publishing, the skill validates:

  • Docstring Completeness: All public functions have docstrings
  • Parameter Documentation: All function parameters are documented
  • Type Annotations: Functions include type hints
  • Example Executability: Generated examples run successfully
  • Link Integrity: Cross-references within documentation are valid
  • AI Confidence: Low-confidence AI-generated sections flagged for review
