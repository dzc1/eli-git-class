Agentic Skill — Documentation Generation

OpenAPI-to-Documentation Pipeline

Automatically generate interactive API reference documentation from OpenAPI/Swagger specifications, keeping documentation in sync with API changes.

Skill Name: openapi-docs-generator Priority: P1 Timeline: Month 2 (Phase 1)

How It Works

  1. Skill monitors repositories for openapi.yaml or swagger.json files
  2. On detection, parses API specification (endpoints, parameters, responses)
  3. Uses AI to generate human-readable descriptions for each endpoint
  4. Produces interactive API reference documentation (HTML/Markdown)
  5. Publishes to Confluence space dedicated to API documentation

Example Output

When the skill detects a change to an OpenAPI specification:

Input:  /api/bookings POST endpoint in openapi.yaml

Output: "Create Hotel Booking" page with:
        - Endpoint description
        - Request parameters (with examples)
        - Response schemas
        - Error codes explained
        - Code samples (Python, JavaScript, cURL)

The generated documentation automatically includes:

  • All endpoint paths with HTTP methods
  • Request/response schemas with type information
  • Authentication and authorization requirements
  • Practical code examples in multiple languages
  • Rate limiting and quota information
  • Error responses with remediation guidance

Benefits

Key Outcomes

API documentation always in sync with OpenAPI spec · Engineers update spec, documentation auto-generates · Reduces manual API reference writing by 90% · Interactive docs improve developer experience · Version-specific documentation for multiple API versions

  • Single Source of Truth: API specification drives documentation; changes automatically propagate
  • Reduced Maintenance: Engineers focus on spec accuracy, not manual doc writing
  • Consistency: All API endpoints documented in standardised format
  • Developer Experience: Try-it-out functionality for interactive API testing
  • Version Control: Documentation versioned alongside API releases

Tooling Integration

The pipeline integrates with industry-standard tools:

  • Rendering: Redoc, Stoplight, or OpenAPI Generator for interactive documentation
  • AI Enhancement: GPT-4, Claude for natural language descriptions and example generation
  • Triggers: GitHub/GitLab webhooks for repository events (push, tag, PR)
  • Publication: Confluence REST API for automated publishing and versioning
  • Validation: Spectacle or OpenAPI Linter to ensure spec compliance

Implementation Details

Trigger Events

The skill is triggered when:

  • OpenAPI specification file is modified
  • New API version tag is pushed to repository
  • Scheduled refresh on daily/weekly basis
  • Manual trigger via CLI or GUI

Quality Validation

Before publishing, the skill validates:

  • OpenAPI spec compliance (using validators)
  • Generated documentation completeness
  • Code example executability
  • Link integrity and cross-references
  • AI confidence scoring for descriptions

Version Management

The skill supports multiple API versions:

  • Current/stable API documentation
  • Beta/preview API documentation
  • Deprecated API versions with migration guidance
  • Feature flag documentation for optional endpoints

Integration with Documentation Operating Model

This skill is part of Layer 1: AI-Assisted Documentation Generation in the broader documentation infrastructure. It works in coordination with:

  • changelog-automator: Detects API changes and triggers documentation refresh
  • version-manager: Maintains parallel documentation for multiple API versions
  • broken-link-detector: Validates generated documentation links
  • missing-metadata-enforcer: Ensures all generated pages include required metadata

In the release workflow, this skill is invoked after a Git tag is pushed (e.g., v2.1.0), executing in parallel with other documentation generation skills to produce comprehensive release documentation.
